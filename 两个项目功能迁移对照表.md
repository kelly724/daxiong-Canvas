# daxiong-Canvas 与 AI-CanvasPro 功能迁移对照表

生成时间：2026-06-09

## 结论

`daxiong-Canvas` 适合作为当前继续开发的主项目。它已经能在 Mac 上跑起来，而且后端接口、即梦 CLI、RunningHub、ComfyUI、ModelScope、API 设置、素材库、提示词库这些能力已经集中在当前项目里。

`AI-CanvasPro` 更适合作为功能参考和架构参考。它的优势不是单个接口更多，而是节点分类、Agent 总控、任务队列、Provider 状态检查、本地 Codex / Suno 工具接入做得更像一个完整产品。

不建议直接把两个项目合并。建议按模块迁移：先搬设计思想，再搬清晰模块，最后才考虑 UI 细节。

## AI-CanvasPro 有哪些节点

AI-CanvasPro 的节点可以分成三类：生成节点、源节点、功能节点。

### 1. 生成节点

| 节点类型 | 中文名 | 用途 | 迁移价值 |
|---|---|---|---|
| `ai-text` | 文本 | 生成文案、脚本、提示词 | 高。可对应当前项目的 `LLM 节点` 和 `提示词节点` |
| `ai-image` | 图像 | 生成图片、海报、角色图、参考图 | 高。可对应当前项目的 `API生成`、`ModelScope生成`、`ComfyUI生成`、`RH生成` |
| `ai-video` | 视频 | 生成短片、转场、动态镜头 | 中高。可对应当前项目的 `视频生成`、`RH生成`、`LTX Director` |
| `ai-audio` | 音频 | 生成配音、音效、音乐 | 高。当前项目缺少真正的一等音频生成节点，值得补 |

### 2. 源节点

源节点不是负责生成，而是负责把素材放进画布，作为后续生成节点的输入。

| 节点类型 | 中文名 | 用途 | 迁移价值 |
|---|---|---|---|
| `source-text` | 源文本 | 放脚本、提示词、文案输入 | 高。可增强当前 `提示词节点` |
| `source-image` | 源图像 | 放参考图、首帧、角色图、素材图 | 高。可对应当前 `上传节点` |
| `source-video` | 源视频 | 放参考视频、剪辑素材、视频输入 | 中高。当前项目已能上传视频，但没有单独源视频节点 |
| `source-audio` | 源音频 | 放配音、BGM、声音参考 | 中高。当前项目能处理音频素材，但没有清晰的音频源节点 |

### 3. 功能节点

功能节点主要是画布工具，不一定直接调用 AI 生成。

| 节点类型 | 中文名 | 用途 | 迁移价值 |
|---|---|---|---|
| `panorama-scene` | 3D导演台 | 3D 场景、人物、机位管理 | 中。功能强，但迁移成本高 |
| `panorama-360` | 360全景图 | 全景图查看、空间关系辅助 | 中。适合后期做空间类视频/场景创作 |
| `storyboard-script` | 分镜脚本 | 镜头表、提示词、节奏规划 | 高。很适合当前画布的视频工作流 |
| `storyboard` | 故事板 / 分镜板 | 把画面切成分镜格或管理分镜 | 中高。适合做短视频生产链路 |
| `collage` | 拼图 | 图片排版、拼图、导出 | 中。当前项目有图片编辑能力，但没有独立拼图节点 |
| `media-clip` | 剪辑 | 音视频剪切整理 | 中。当前项目可先不搬，等视频链路稳定后再做 |
| `web-preview` | 浏览器 | 画布内打开网页预览 | 低到中。不是当前最急功能 |
| `web-reference-card` | 网页参考卡 | 保存网页引用、网页素材卡片 | 中。适合做资料收集型画布 |
| `comment-note` | 评论 / 便签 | 画布批注、说明、协作备注 | 中。容易做，提升可用性 |
| `scene-detection` | 场景检测 | 从视频/素材里识别镜头或场景 | 中。适合后期视频工作流 |
| `group` | 分组 | 把节点组合管理 | 当前项目已有类似能力 |
| `debug` | 调试节点 | 查看 payload、任务状态、调试信息 | 中。开发期很有用，普通用户可隐藏 |

## 迁移对照表

| AI-CanvasPro 功能 | 当前 daxiong-Canvas 对应能力 | 建议怎么迁移 | 优先级 |
|---|---|---|---|
| `ai-text` 文本生成 | `llm`、`prompt`、`smart-prompt` | 不必新建大模块，先把当前 LLM 节点包装成“文本生成节点”体验 | P1 |
| `source-text` 源文本 | `prompt`、`smart-prompt` | 短期复用提示词节点；长期可增加“源文本”概念 | P1 |
| `ai-image` 图像生成 | `generator`、`msgen`、`comfy`、`rh` | 当前项目能力更强，借 AI-CanvasPro 的模型清单/状态检查/节点分类即可 | P1 |
| `source-image` 源图像 | `image`、`smart-image` | 当前已支持，建议增加“源图片 / 结果图片 / 参考图片”的角色标记 | P1 |
| `ai-video` 视频生成 | `video`、`rh`、`ltxDirector`、`comfy` | 保留当前视频接口，借 AI-CanvasPro 的视频节点交互和 Provider 就绪检查 | P2 |
| `source-video` 源视频 | `image` 节点里承载 video kind | 短期继续用媒体节点；长期拆出源视频节点，让新手更容易理解 | P2 |
| `ai-audio` 音频生成 | 当前没有一等生成节点 | 从 AI-CanvasPro 借 Suno / RunningHub 音频思路，新增“音频生成节点” | P1 |
| `source-audio` 源音频 | 当前可上传音频，但不是独立节点 | 新增“源音频节点”，给视频、数字人、配音工作流使用 | P2 |
| `storyboard-script` 分镜脚本 | 当前主要靠提示词/LLM 手动写 | 高价值迁移。适合先做“分镜脚本 -> 图片节点 -> 视频节点”的生产链 | P1 |
| `storyboard` 故事板 | 当前无明确故事板节点 | 后续把多张图/多镜头组织成故事板，服务短视频流程 | P2 |
| `collage` 拼图 | 当前有图片编辑、网格切分、素材库 | 可后置。当前项目先补 Agent 和音频更重要 | P3 |
| `panorama-scene` 3D导演台 | `ltxDirector` 有部分导演台味道，但不是 3D 场景 | 功能复杂，后期再迁移；不建议第一轮做 | P3 |
| `panorama-360` 360 全景图 | 当前没有对应一等节点 | 后期做空间场景、VR/全景素材时再考虑 | P3 |
| `media-clip` 剪辑 | 当前有视频/音频素材处理痕迹，但没有剪辑节点 | 等视频链路稳定后再做 | P3 |
| `web-preview` 浏览器节点 | 当前无 | 可作为资料采集/网页参考功能，非当前核心 | P3 |
| `web-reference-card` 网页参考卡 | 当前无 | 适合做资料卡片，但不影响生成主流程 | P3 |
| `comment-note` 评论便签 | 当前缺少轻量备注节点 | 容易迁移，适合中期补上 | P2 |
| `debug` 调试节点 | 当前主要靠日志和接口返回 | 开发期可加，普通用户隐藏 | P2 |
| Agent 总控 / Flow 面板 | 当前只有级联运行，没有完整 Agent 规划层 | 这是最值得借的功能。先做只读规划，再做节点创建，再做确认执行 | P0 |
| 任务确认队列 | 当前有图片/视频任务，但没有统一队列 | 借 AI-CanvasPro 的 preview-only 思路，防止误触批量生成消耗额度 | P0 |
| Provider 状态检查 | 当前更多是 API 设置和连接测试 | 借 AI-CanvasPro 的“是否安装、是否登录、是否支持能力”检查 | P1 |
| Codex CLI 生图 | 当前没有一等 Codex 节点 | 借 `codex_cli_service` 和 `codexCliImageManifest`，作为本机生图 Provider | P1 |
| Suno CLI 音乐 | 当前没有一等音频生成 | 借 `suno_cli_service` 和 `sunoCliAudioManifest`，作为音频节点基础 | P2 |
| 模型 Manifest 注册 | 当前大量逻辑写在大 JS / 后端条件分支里 | 不建议一次性全迁。先给 Codex/Suno/即梦做小型 Manifest 试点 | P2 |

## 推荐迁移顺序

### 第 0 阶段：只做设计映射

目标：不写功能代码，只确定节点怎么对应。

- `ai-text` 对应当前 `llm`
- `ai-image` 对应当前 `generator` / `msgen` / `comfy` / `rh`
- `ai-video` 对应当前 `video` / `rh` / `ltxDirector`
- `ai-audio` 是当前缺口，需要新建
- `source-*` 可以短期复用当前媒体节点，长期再拆独立节点

### 第 1 阶段：Agent 只读规划

目标：用户输入一句话，Agent 只输出计划，不创建节点、不调用接口。

示例：

```text
用户：帮我做一个 30 秒赛博朋克短片
Agent 输出：
1. 创建分镜脚本
2. 创建 6 个图片节点
3. 每张图生成一个视频节点
4. 创建一个 BGM 音频节点
5. 最后进入确认队列
```

这一阶段风险最低，也最容易验证。

### 第 2 阶段：Agent 创建节点，但不生成

目标：Agent 可以在当前画布上创建 `prompt`、`generator`、`video`、`output`、`loop` 等节点，并连线。

注意：这一阶段仍然不自动提交生成任务。

### 第 3 阶段：确认队列

目标：所有生成动作先进队列，让用户确认。

这个功能很重要，因为以后一个 Agent 可能一次创建 20 个节点。如果没有确认队列，很容易误触消耗 API 额度。

### 第 4 阶段：补 Codex CLI 生图

目标：把 AI-CanvasPro 里的 Codex CLI 生图能力迁到当前项目。

当前项目可以新增：

- `/api/codex-cli/status`
- `/api/codex-cli/imagegen`
- 画布里的 `Codex 生图` Provider

### 第 5 阶段：补音频节点

目标：新增一等音频节点。

可以分两步：

1. 先做 `source-audio`，只上传/引用音频。
2. 再做 `ai-audio`，接 Suno 或 RunningHub 音频工作流。

## 不建议第一轮迁移的内容

| 功能 | 原因 |
|---|---|
| 整个 Electron 桌面壳 | 当前项目 Mac 启动问题可以用 `.command` 解决，没必要先做桌面壳 |
| 整套 AI-CanvasPro UI | 两边数据结构不同，直接搬容易坏 |
| 3D导演台 / 360全景 | 功能大、迁移成本高，不是当前核心痛点 |
| 会员/订阅/权限门槛 | 对当前本地工具没有必要，反而会让新手困惑 |
| 混淆压缩过的核心代码 | 可读性差，不适合直接复制 |
